## 1. 概述

知识库助手是一个基于本地大语言模型（LLM）的 Obsidian 插件，能够：

- **智能问答**：基于知识库内容回答问题，自动引用来源
- **知识检索**：结合向量搜索和关键词搜索，精准定位相关内容
- **自动索引**：监控文件变更，自动更新向量数据库
- **内容处理**：自动分类、实体识别、链接推荐
- **结构化输入**：引导式创建实验记录、文献笔记等标准文档
- **快速捕获**：收件箱模式，快速记录灵感和碎片知识

### 技术栈

| 组件 | 技术 |
|------|------|
| 前端插件 | TypeScript + Obsidian API |
| 后端服务 | Python + FastAPI |
| 向量数据库 | ChromaDB |
| 嵌入模型 | nomic-embed-text (Ollama) |
| 大语言模型 | qwen2.5:7b (Ollama) |
| 文件监控 | watchdog |

## 3. 安装与启动

### 3.1 前置条件

1. **Obsidian**：版本 ≥ 1.0.0
2. **Python**：版本 ≥ 3.9
3. **Ollama**：已安装并运行
4. **模型**：已拉取以下模型
   ```bash
   ollama pull nomic-embed-text
   ollama pull qwen2.5:7b
   ```

### 3.2 安装插件

1. 将 `kb-plugin` 文件夹复制到 Obsidian 插件目录：
   ```
   <Vault路径>/.obsidian/plugins/kb-plugin/
   ```

2. 在 Obsidian 中启用插件：
   - 打开设置 → 第三方插件 → 已安装插件
   - 找到"知识库助手"并启用

### 3.3 启动后端

**方式一：自动启动（推荐）**

插件启用后会自动检测并启动后端服务。确保：
- Python 已安装且在 PATH 中
- Ollama 正在运行

**方式二：手动启动**

```bash
cd <Vault路径>/00_系统/kb-backend
python main.py
```

### 3.4 验证安装

1. 点击左侧边栏的机器人图标 🤖
2. 查看状态栏是否显示"向量: xxx | 对话: xxx"
3. 点击"状态"按钮查看详细信息

## 6. 功能详解

### 6.1 智能问答（核心功能）

**使用方法**：
1. 在输入框输入问题
2. 按 `Enter` 或点击"发送"
3. AI 基于知识库内容实时生成回答
4. 回答下方显示参考来源，点击可跳转

**示例**：
```
用户：CRISPRoff是什么？
AI：CRISPRoff是一种利用CRISPR系统进行表观遗传沉默的方法，
    通过表达dCas9融合蛋白来抑制目标基因的转录和DNA甲基化...
📎 来源：
    • CRISPRoff实验 (99%)
    • 概念卡片_CRSIPRoff (95%)
🔗 相关概念：[[dCas9]] [[KRAB]] [[DNMT3A]]
```

**特点**：
- 流式输出：回答逐字显示，无需等待全部生成
- 来源引用：每条回答标注参考文件和相关度
- 概念链接：自动识别并推荐相关概念卡片
- 自动保存：问答自动保存到 `01_输入区/对话记录/`

**换行**：`Shift + Enter` 可在输入框中换行

### 6.4 冲突检测

**位置**：工具栏 → `冲突` 按钮

**功能**：
- 自动扫描概念卡片之间的矛盾信息
- 检测数据不一致（如数值、参数不同）
- 发现模糊或可能引起误解的表述

**显示结果**：
- ✅ 未发现冲突
- ⚠️ 发现 N 个冲突（显示冲突详情）

**触发时机**：
- 启动 30 秒后自动扫描
- 手动点击"冲突"按钮触发
- 缓存 1 小时，避免重复扫描

## 10. 故障排除

### 10.1 后端未连接

**症状**：状态栏显示"后端未连接"

**解决方案**：
1. 检查 Ollama 是否运行：`ollama list`
2. 检查后端是否启动：`curl http://localhost:8000/api/status`
3. 手动启动后端：
   ```bash
   cd <Vault路径>/00_系统/kb-backend
   python main.py
   ```

### 10.2 搜索无结果

**症状**：提问后返回"知识库中未找到相关信息"

**解决方案**：
1. 检查向量数是否为 0
2. 如果为 0，需要重建索引：
   ```bash
   curl -X POST http://localhost:8000/api/index
   ```
3. 等待索引完成（可能需要 15-20 分钟）

### 10.3 响应速度慢

**症状**：AI 回答生成很慢

**原因**：
- CPU 模式运行 LLM（无 GPU 加速）
- 知识库内容较多

**解决方案**：
- 等待（首次加载模型需要时间）
- 考虑使用更小的模型
- 如有 NVIDIA GPU，配置 Ollama 使用 GPU

### 10.4 ChromaDB 损坏

**症状**：启动时报错 "Error loading hnsw index"

**解决方案**：
1. 停止后端服务
2. 删除 `kb-backend/chroma_db` 目录
3. 重启后端（会自动重新索引）

### 10.5 插件设置不生效

**症状**：修改设置后没有效果

**解决方案**：
1. 重启 Obsidian
2. 重新启用插件
3. 检查设置是否正确保存
